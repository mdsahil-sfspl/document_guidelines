Configure Git with Your Details
‚ö†Ô∏è Important: Type these commands exactly as shown. Replace the example name and email with YOUR actual details.

Example: If your name is Priya:

  • Type the command in terminal
  • Press Enter

You should see your name and email in the output. ‚úÖ

4
Generate SSH Key (Secure Connection)
üìå What is SSH Key? SSH key is like a digital password that lets your computer securely connect to GitHub without typing your password every time.
4.1 Create SSH Key
ssh-keygen -t ed25519 -C "your-email@sampurna.co.in"

Example:

ssh-keygen -t ed25519 -C "priya@sampurna.co.in"
‚ö†Ô∏è What Will Happen Next:
  • Terminal will ask: "Enter file in which to save the key"
  • Just press Enter (accept default location)
  • Terminal will ask: "Enter passphrase"
  • Just press Enter (leave empty for no passphrase)
  • Terminal will ask again: "Enter same passphrase again"
  • Just press Enter again
  • You'll see a "key fingerprint" message - That's success! ‚úÖ
4.2 View Your SSH Key
cat ~/.ssh/id_ed25519.pub

This will display a long string starting with ssh-ed25519. This is your SSH public key.

4.3 Copy the SSH Key
  • The key will appear in terminal (long text starting with ssh-ed25519)
  • Use your mouse to select ALL the text (from ssh-ed25519 to your email)
  • Right-click and select "Copy"
  • OR press Ctrl + Shift + C
üí° Pro Tip: Keep this terminal window open! You'll need this key in the next step.
5
Add SSH Key to GitHub Account
5.1 Open GitHub Settings
  • Go to https://github.com in your browser
  • Login if not already logged in
  • Click on your profile picture (top-right corner)
  • Click "Settings" from dropdown menu
5.2 Navigate to SSH Keys
  • On the left sidebar, scroll down
  • Find and click "Access" section
  • Click on "SSH and GPG keys"
5.3 Add New SSH Key
  • Click green button "New SSH key"
  • Title: Give it a name (e.g., "My Ubuntu Laptop" or "Office Computer")
  • Key: Paste the SSH key you copied earlier
  • Press Ctrl + V to paste
  • Click green button "Add SSH key"
  • GitHub may ask for your password - enter it
üéâ Success! Your computer is now securely connected to GitHub. You won't need to enter password every time you push code!

Section 5: Setting Up Python Environment

1
Create Virtual Environment (venv)
üìå What is Virtual Environment? A venv is like a separate container for your project's Python packages. It keeps this project's packages separate from other projects, preventing conflicts.
‚ö†Ô∏è IMPORTANT: Make sure you're INSIDE your project folder before creating venv!
# Navigate to project folder
cd ~/Projects/chatbot-builder-platform_yourName

# Check you're in the right place (should see README.md)
ls

# Create virtual environment named 'venv'
python3 -m venv venv

# This will take 10-20 seconds... wait for command prompt to return
Verify venv was Created
# List files - you should now see 'venv' folder
ls
# Output should include: README.md venv/
2
Activate Virtual Environment
üìå Why Activate? Activating tells terminal to use the Python and packages from THIS project's venv, not the system's Python.
# Activate venv
source venv/bin/activate

# Press Enter
‚úÖ How to Know It's Activated: Your terminal prompt will change. You'll see (venv) at the beginning:
(venv) user@computer:~/Projects/chatbot-builder-platform_yourName$
üí° Remember:
  • Activate venv: source venv/bin/activate
  • Deactivate venv: deactivate
  • You need to activate venv EVERY TIME you open a new terminal!

-- Chatbot Builder Platform Database Schema
-- Created by: Sampurna Financial Team
-- Date: 2025-01-15

-- Users Table
CREATE TABLE IF NOT EXISTS users (
user_id INT PRIMARY KEY AUTO_INCREMENT,
username VARCHAR(100) UNIQUE NOT NULL,
email VARCHAR(255) UNIQUE NOT NULL,
password_hash VARCHAR(255) NOT NULL,
full_name VARCHAR(200),
role ENUM('admin', 'developer', 'viewer') DEFAULT 'developer',
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
is_active BOOLEAN DEFAULT TRUE
);

-- Chatbots Table
CREATE TABLE IF NOT EXISTS chatbots (
chatbot_id INT PRIMARY KEY AUTO_INCREMENT,
chatbot_name VARCHAR(200) NOT NULL,
description TEXT,
created_by INT NOT NULL,
status ENUM('draft', 'active', 'inactive') DEFAULT 'draft',
welcome_message TEXT,
fallback_message TEXT,
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
FOREIGN KEY (created_by) REFERENCES users(user_id) ON DELETE CASCADE
);

-- Flows Table (Conversation flows)
CREATE TABLE IF NOT EXISTS flows (
flow_id INT PRIMARY KEY AUTO_INCREMENT,
chatbot_id INT NOT NULL,
flow_name VARCHAR(200) NOT NULL,
flow_type ENUM('linear', 'branching', 'ai_powered') DEFAULT 'linear',
flow_json JSON,
is_default BOOLEAN DEFAULT FALSE,
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
FOREIGN KEY (chatbot_id) REFERENCES chatbots(chatbot_id) ON DELETE CASCADE
);

-- Nodes Table (Flow nodes/steps)
CREATE TABLE IF NOT EXISTS nodes (
node_id INT PRIMARY KEY AUTO_INCREMENT,
flow_id INT NOT NULL,
node_type ENUM('message', 'question', 'condition', 'action', 'api_call') NOT NULL,
node_content TEXT,
node_config JSON,
position_x INT DEFAULT 0,
position_y INT DEFAULT 0,
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
FOREIGN KEY (flow_id) REFERENCES flows(flow_id) ON DELETE CASCADE
);

-- Conversations Table
CREATE TABLE IF NOT EXISTS conversations (
conversation_id INT PRIMARY KEY AUTO_INCREMENT,
chatbot_id INT NOT NULL,
user_identifier VARCHAR(255),
session_id VARCHAR(255) UNIQUE NOT NULL,
started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
ended_at TIMESTAMP NULL,
status ENUM('active', 'completed', 'abandoned') DEFAULT 'active',
FOREIGN KEY (chatbot_id) REFERENCES chatbots(chatbot_id) ON DELETE CASCADE
);

-- Messages Table
CREATE TABLE IF NOT EXISTS messages (
message_id INT PRIMARY KEY AUTO_INCREMENT,
conversation_id INT NOT NULL,
sender ENUM('user', 'bot') NOT NULL,
message_text TEXT NOT NULL,
message_type ENUM('text', 'image', 'button', 'quick_reply') DEFAULT 'text',
metadata JSON,
sent_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
FOREIGN KEY (conversation_id) REFERENCES conversations(conversation_id) ON DELETE CASCADE
);

-- Integrations Table
CREATE TABLE IF NOT EXISTS integrations (
integration_id INT PRIMARY KEY AUTO_INCREMENT,
chatbot_id INT NOT NULL,
integration_type VARCHAR(50) NOT NULL,
config JSON,
is_enabled BOOLEAN DEFAULT TRUE,
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
FOREIGN KEY (chatbot_id) REFERENCES chatbots(chatbot_id) ON DELETE CASCADE
);

-- Analytics Table
CREATE TABLE IF NOT EXISTS analytics (
analytics_id INT PRIMARY KEY AUTO_INCREMENT,
chatbot_id INT NOT NULL,
event_type VARCHAR(100),
event_data JSON,
recorded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
FOREIGN KEY (chatbot_id) REFERENCES chatbots(chatbot_id) ON DELETE CASCADE
);

-- Create indexes for better performance
CREATE INDEX idx_chatbot_status ON chatbots(status);
CREATE INDEX idx_conversation_session ON conversations(session_id);
CREATE INDEX idx_message_conversation ON messages(conversation_id);
CREATE INDEX idx_analytics_chatbot ON analytics(chatbot_id);

# ============================================

# Create database
mysql -u root -p -e "CREATE DATABASE chatbot_db;"

# Run schema file
mysql -u root -p chatbot_db < database/schema.sql

# Run schema file
psql -U postgres -d chatbot_db -f database/schema.sql

# Login to database
psql -U postgres -d chatbot_db

Problem: Accidentally Committed venv
Symptoms: Git status shows 1000+ files, git push takes forever, venv/ appears in git ls-files
# Solution:

# 1. Remove venv from Git (keeps local copy)
git rm -r --cached venv/

# 2. Make sure .gitignore has venv/
echo "venv/" >> .gitignore

# 3. Commit the removal
git add .gitignore
git commit -m "Remove venv from repository and update .gitignore"

# 4. Push
git push origin main

# 5. Verify
git ls-files | grep venv
# Should be empty ‚úÖ
2
Problem: Permission Denied (SSH)
Error: "Permission denied (publickey)" when pushing/pulling
# Solution:

# 1. Check if SSH key exists
ls -la ~/.ssh/
# Should see id_ed25519 and id_ed25519.pub

# 2. Test SSH connection
ssh -T git@github.com
# Should say: "Hi username! You've successfully authenticated"

# 3. If not working, regenerate SSH key
ssh-keygen -t ed25519 -C "your-email@sampurna.co.in"

# 4. Copy new key
cat ~/.ssh/id_ed25519.pub

# 5. Add to GitHub (Settings ‚Üí SSH Keys)

# Solution:

# 1. Open conflicted file in editor
nano filename.py

# <<<<<<< HEAD
